The Incident

A production incident was triggered by a configuration change. The operations team needed to identify exactly when a specific timeout value of 270 was introduced into config.json, so they could correlate it with other changes deployed at the same time.

Investigation Methodology

Step 1 ‚Äî Search Commit History
Used git log -S "270" to find commits where the string "270" was added or removed in config.json.
Step 2 ‚Äî Inspect the Diff
Ran git show 742a75b to examine the exact changes made ‚Äî confirming the timeout was changed from 420 ‚Üí 270.
Step 3 ‚Äî Identify Parent Commit
Used git rev-parse --short=7 742a75b^ to retrieve the 7-character short hash of the parent commit.
